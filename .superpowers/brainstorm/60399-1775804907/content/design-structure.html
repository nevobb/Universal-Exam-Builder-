Design Section 1 — File Structure & Integration

How the viewer slots into the existing project

File tree — changes highlighted
app/
  index.html  ← add KaTeX CDN links here
  Architecture.jsx  ← add "Exam Viewer" tab + import ExamViewer
  src/
    viewer/  ← NEW directory
      usePyodide.js  ← lazy init hook + micropip installer
      DiagramViewer.jsx  ← runs Python → renders SVG
      QuestionCard.jsx  ← renders one question (LaTeX + diagram + solution toggle)
      ExamViewer.jsx  ← page: JSON input + question list + loading overlay
Integration point in Architecture.jsx
// 1. One import at top of Architecture.jsx
import ExamViewer from './src/viewer/ExamViewer';

// 2. Add tab to existing tab list
{ id: 'viewer', label: '📋 צפייה במבחן' }

// 3. Render in existing screen switch
{activeTab === 'viewer' && <ExamViewer sharedData={examData} />}

Key point

The window.__GEMINI_EXAM_DATA__ slot already exists in index.html. The viewer will poll this + receive data via a prop from Architecture.jsx, so both injection paths work with zero index.html changes beyond adding the KaTeX CDN link.
